# Action Workflow Reference

## Safety Gate

```text
用户明确确认真实动作？
  ├─ 否 -> 仅允许 previewTask：填表、截图、返回摘要，不提交
  └─ 是 -> 保留审批记录 -> dry-run/预览截图 -> 二次确认 -> 真实执行
```

`executorDryRun` 是运行态安全开关；调试、预览和文档更新都不得自行修改它。

## 通用执行流程

```text
接收 action
  -> 校验 action 类型、任务 ID、参数和审批状态
  -> 建立/复用 CDP 连接
  -> 导航至正确的任务中心或创建页
  -> 等页面身份、目标表格和目标行的 DOM 信号
  -> beforeScreenshot
  -> 定位精确控件并操作
  -> 等确认弹窗/表单回显/结果信号
  -> afterScreenshot
  -> 返回 { ok, step, screenshots, summary }
```

失败路径：任一步 DOM 信号、页面身份或确认回显不成立，停止后续点击，保存 `failureScreenshot`，返回失败步骤与诊断；不可把“页面打开”当成“动作成功”。

## `create_boost_task`

```text
任务中心 -> 调控 -> 素材追投 -> 新建追投
  -> 选择子模式
      ├─ 放量：选择素材 -> 填预算/时长/ROI（如页面要求）
      ├─ 控成本-支付 ROI：选择素材 -> 填支付 ROI/预算/时长
      └─ 控成本-直播出价：选择素材 -> 填出价/预算/时长
  -> 校验每个表单值回显
  -> previewTask 截图，或真实执行前 dry-run 截图
  -> 仅在二次确认后点击提交
  -> 等待提交成功/任务行出现 -> afterScreenshot
```

直播间画面追投使用直播间画面作为素材源时，仍应先验证页面已选中对应来源和参数回显；不得因为没有素材 ID 而跳过表单校验。

## `create_oneclick_task`

```text
任务中心 -> 调控 -> 一键起量 -> 新建
  -> 选择“直播间购买”
  -> 读取直播大屏截图作为素材来源证据
  -> 填预算和时长
  -> 检查表单回显
  -> previewTask 截图 / dry-run 截图
  -> 二次确认后提交
  -> 等成功提示或任务列表新行 -> afterScreenshot
```

若“直播间购买”或直播间画面控件未出现，返回定位失败和截图，不得猜测性点击相邻按钮。

## 现有任务快速路径

```text
pause_task / end_task / increase_task_budget / decrease_task_budget /
extend_task_duration / change_roi_target
  -> 任务中心 -> 调控 -> 对应分类页
  -> 以任务 ID 定位唯一任务行
  -> beforeScreenshot
  -> 点击该行内对应操作
  -> 填参数（如有）并检查弹窗回显
  -> 确认弹窗
  -> 等状态、预算、时长或 ROI 更新
  -> afterScreenshot
```

暂停和结束是不可逆或影响投放的动作。即便按钮定位正确，也必须先满足人工确认、动作 ID 去重和冷却规则。

## `previewTask` 与真实执行

| 阶段 | 允许操作 | 禁止操作 | 产物 |
|---|---|---|---|
| `previewTask` | 导航、选择、填表、读取回显、截图 | 点击提交/确认投放 | 表单摘要、预览截图 |
| dry-run | 模拟完整路径、填写、截图 | 最终提交、变更线上任务 | dry-run 结果、截图 |
| confirmed execute | 前两阶段通过后的精确点击 | 绕过审批或二次确认 | 前后截图、结果、失败截图 |

## SPA、反爬与重试

- 每次导航后等待 URL、可见容器、目标行或表单标题等 DOM 信号，不用固定时长判断页面完成。
- 在非提交操作之间加入短随机延迟，避免连续机械点击；延迟不能代替 DOM 回显验证。
- 页面 loading、切换分类或提交回显未完成时继续轮询 DOM；超时后最多按既有策略重试。
- 重试前重新确认账号、页面 URL 和目标任务 ID。任务行不唯一、页面已漂移或元素不可见时直接失败。
- 对失败、页面改版或登录失效保留截图与步骤日志，供更新精确选择器；不降级为宽泛选择器。
